# llamadeck

Маленькая Rust-обёртка над **llama.cpp**: веб-чат в браузере, режим CLI и живая
статистика (запросы / токены / средняя скорость).

Архитектура нарочно простая: инференс делает родной `llama-server` из llama.cpp
(OpenAI-совместимый HTTP API), а llamadeck — это слой сверху (UI + CLI + статистика).
Весь код общения с движком изолирован в `src/backend.rs` — позже его можно
заменить на FFI (`llama-cpp-2`), чтобы собрать всё в один бинарник, не трогая
остальное.

```
+-----------+      HTTP/JSON      +--------------+      FFI/llama.cpp
|  llamadeck| <-----------------> | llama-server | ------------------> модель .gguf
| (Rust)    |  /v1/chat/...       | (из llama.cpp)|
+-----------+                     +--------------+
   |  ^
веб|  |CLI
```

## Что нужно один раз

1. **Rust** — установить с https://rustup.rs (на Windows скачается `rustup-init.exe`).
2. **llama.cpp с `llama-server`**. Самый простой путь на Windows — скачать готовые
   бинарники со страницы релизов проекта `ggml-org/llama.cpp` на GitHub
   (ищи архив вида `llama-bXXXX-bin-win-...`). Внутри будет `llama-server.exe`.
3. **Модель в формате .gguf** — например `Qwen3-30B-A3B-Instruct-2507` (квант
   `Q4_K_M`, ~18.5 ГБ) с Hugging Face, репозиторий `unsloth`. Это MoE: всего 30B,
   но активны ~3B на токен — подходит для слабого GPU + 32 ГБ RAM.

## Запуск в один терминал (рекомендуется)

llamadeck сам поднимет `llama-server` и погасит его при выходе — нужно лишь указать
путь к модели через `--model-path`:

```
cargo run --release -- --model-path models\Qwen3-30B-A3B-Instruct-2507-Q4_K_M.gguf serve
```
Открой в браузере http://127.0.0.1:3000

Полезные флаги авто-запуска (все необязательные):
- `--llama-bin llama\llama-server.exe` — путь к движку, если он не в PATH;
- `--ngl 0` — сколько слоёв выгрузить на GPU. На T2000 (4 ГБ) начинай с малого;
  `0` — целиком на CPU (надёжно, для A3B вполне терпимо);
- `--ctx 4096` — размер контекста;
- `--engine-port 8080` — порт, на котором поднимать движок.

Режим терминала вместо браузера — то же самое, но `chat`:
```
cargo run --release -- --model-path models\Qwen3-30B-A3B-Instruct-2507-Q4_K_M.gguf chat
```

## Запуск в два терминала (свой llama-server)

Если хочешь управлять движком сам — запусти его отдельно и подключись по `--upstream`:

**Терминал 1 — движок:**
```
llama-server.exe -m models\Qwen3-30B-A3B-Instruct-2507-Q4_K_M.gguf --port 8080 -c 4096 -ngl 0 --jinja
```
(`--jinja` нужен для агентского режима — разбор вызовов инструментов.)

**Терминал 2 — llamadeck (без `--model-path` → подключается к готовому):**
```
cargo run --release -- serve
cargo run --release -- --upstream http://127.0.0.1:9090 serve   # другой адрес движка
```

## Агентский режим (вкладка «Агент»)

Кроме обычного чата есть режим **агента**: модель сама вызывает инструменты, а ты
контролишь, что ей разрешено, и видишь каждый её шаг. Идея взята из проекта
[chebupelka](https://github.com/alexey-goloburdin/chebupelka), сверху добавлены
разрешения и наблюдаемость.

Как пользоваться: открой вкладку «Агент», отметь галочками разрешённые инструменты,
опиши задачу и нажми «Запустить». В ленте появятся все вызовы инструментов с
аргументами и их результаты, а в конце — итоговый ответ.

Инструменты:
- `read_file` — прочитать файл (безопасно);
- `list_dir` — список файлов в папке (безопасно);
- `write_file` — создать/перезаписать файл (**меняет диск**);
- `run_command` — выполнить команду в системе (**мощно и опасно**).

Разрешения — «список заранее»: агенту предъявляются **только отмеченные**
инструменты. По умолчанию включены лишь чтение (`read_file`, `list_dir`); запись и
команды включай осознанно. Инструменты работают относительно текущей рабочей папки.

**Режим прав** (выпадающий список):
- **спрашивать каждый раз** — агент останавливается перед каждым вызовом, а ты
  жмёшь «Разрешить»/«Запретить» прямо в ленте (human-in-the-loop);
- **выполнять автоматически** — разрешённые инструменты выполняются без вопросов.

Переключается на лету; запрет конкретного вызова возвращается модели как «отклонено»,
и она продолжает с учётом этого.

> Требует tool-calling на стороне движка. При авто-запуске (`--model-path`) флаг
> `--jinja` добавляется сам. Если поднимаешь `llama-server` вручную — добавь `--jinja`.

## История диалогов и статистика

- **История** (тулбар в чате): диалог автоматически сохраняется на диск после
  каждого ответа модели (в `data/conversations/`, по JSON-файлу на диалог). Кнопка
  «+ Новый» начинает чистый диалог, выпадающий список — открывает сохранённый.
- **Статистика** (вкладка): счётчики (запросы, токены, средняя скорость, аптайм) и
  **график скорости** последних запросов (ток/с) — рисуется на canvas, без внешних
  библиотек.

## Эндпоинты

| Метод | Путь                | Назначение                         |
|-------|---------------------|------------------------------------|
| GET   | `/`                 | страница чата                      |
| POST  | `/api/chat`         | запрос → полный ответ (JSON)       |
| POST  | `/api/chat/stream`  | потоковый ответ (SSE, токен-за-токеном) |
| GET   | `/api/stats`        | счётчики в JSON                    |
| GET   | `/api/agent/tools`  | список инструментов агента (для галочек) |
| POST  | `/api/agent/stream` | запуск агента, поток событий (SSE) |
| POST  | `/api/agent/approve`| подтвердить/отклонить вызов (режим «спрашивать») |
| GET   | `/api/history`      | список сохранённых диалогов        |
| POST  | `/api/history`      | сохранить/обновить диалог          |
| GET   | `/api/history/{id}` | загрузить диалог                   |
| DELETE| `/api/history/{id}` | удалить диалог                     |

Тело для чата:
```json
{ "messages": [ { "role": "user", "content": "привет" } ] }
```
