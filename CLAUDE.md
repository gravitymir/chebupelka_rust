## Что это за проект

**llamadeck** — небольшая Rust-обёртка над **llama.cpp** для локального запуска LLM.
Даёт три вещи поверх движка инференса: веб-чат в браузере, режим CLI и живую
статистику (запросы / токены / средняя скорость).

Расположение на машине пользователя: `C:\Users\CEX\Documents\rust\LocalAI`
(ОС — Windows). Железо: Dell Precision 7550, Intel i7-10850H, 32 ГБ RAM,
NVIDIA Quadro **T2000 (всего 4 ГБ VRAM)** — это ключевое ограничение, держи в уме
при любых советах про размеры моделей и `-ngl`.

## Архитектурное решение (важно — не ломай без обсуждения)

Инференс делает НЕ наш код, а родной `llama-server` из llama.cpp по
OpenAI-совместимому HTTP API (`POST /v1/chat/completions`). llamadeck — это слой
сверху. Так сделано осознанно: вся тяжёлая C++‑часть остаётся в готовом
llama-server, а наш код — чистый Rust, собирается без боли.

```
llamadeck (Rust) --HTTP/JSON--> llama-server (llama.cpp) --> модель .gguf
   |  веб (axum) + CLI + статистика
```

Весь код общения с движком ИЗОЛИРОВАН в `src/backend.rs`. Это сделано специально,
чтобы будущий переезд на FFI (крейт `llama-cpp-2`, один бинарник без отдельного
llama-server) затронул только этот файл. Не размазывай вызовы движка по другим
модулям.

## Карта файлов

- `src/main.rs` — точка входа, CLI на clap. Подкоманды: `serve` (веб) и `chat` (терминал).
- `src/backend.rs` — слой инференса: `complete()` (полный ответ) и `stream()` (потоковый). Тут и только тут — знание про llama-server.
- `src/server.rs` — axum: роуты `/`, `/api/chat`, `/api/chat/stream` (SSE), `/api/stats`.
- `src/stats.rs` — атомарные счётчики в памяти + `snapshot()`.
- `assets/index.html` — страница чата (встраивается через `include_str!`), стриминг через fetch+reader, опрос статистики раз в секунду.
- `README.md` — инструкция для человека (установка, запуск).

## Как собрать и запустить

```
cargo run --release -- serve          # веб на http://127.0.0.1:3000
cargo run --release -- chat           # чат в терминале
cargo run --release -- --upstream http://127.0.0.1:9090 serve   # другой адрес движка
```
Перед этим в отдельном терминале должен крутиться движок:
```
llama-server.exe -m <путь>.gguf --port 8080 -c 4096 -ngl 10
```
(`-ngl` начинать с 8–12 из-за 4 ГБ VRAM).

## Нюанс с версиями зависимостей

В `Cargo.toml` стоят пины `clap = "=4.5.4"` и `url = "=2.5.0"`. Это артефакт
сборочной среды, где был **Rust 1.75**. На машине пользователя Rust свежий (rustup),
поэтому пины НЕ нужны. Можно безопасно вернуть `clap = "4"` и убрать строку с `url`,
затем `cargo update`. Сделай это, если потребуются свежие фичи clap.

## Соглашения в коде

- Комментарии и тексты для пользователя — на русском.
- Ошибки — через `anyhow`, с понятными человеку сообщениями (см. `backend.rs`).
- Не добавляй localStorage/внешние UI-фреймворки в `index.html` без необходимости — страница нарочно минимальная и без зависимостей.
- HTTP-only (без TLS): общаемся только с localhost, поэтому reqwest собран без TLS-бэкенда. Не включай TLS без причины.

## Дорожная карта (что пользователь хочет дальше — уточни приоритет)

1. Авто-запуск `llama-server` дочерним процессом из llamadeck (`tokio::process::Command`) + health-check, чтобы был один терминал, а не два.
2. Кнопка «стоп» — отмена генерации (пробросить cancel в SSE-стрим).
3. Сохранение истории диалогов на диск.
4. График скорости на странице статистики.
5. Переезд на FFI (`llama-cpp-2`) → один бинарник. Это самый крупный шаг; меняем только `backend.rs`, на Windows понадобится C++‑тулчейн и cmake.

пользователя, какой пункт дорожной карты берём.
