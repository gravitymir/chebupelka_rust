## Что это за проект

**llamadeck** — небольшая Rust-обёртка над **llama.cpp** для локального запуска LLM.
Даёт три вещи поверх движка инференса: веб-чат в браузере, режим CLI и живую
статистику (запросы / токены / средняя скорость).

Расположение на машине пользователя: `C:\Users\CEX\Documents\rust\LocalAI`
(ОС — Windows). Железо: Dell Precision 7550, Intel i7-10850H, 32 ГБ RAM,
NVIDIA Quadro **T2000 (всего 4 ГБ VRAM)** — это ключевое ограничение, держи в уме
при любых советах про размеры моделей и `-ngl`.

## Архитектурное решение (важно — не ломай без обсуждения)

Инференс делает НЕ наш код, а родной `llama-server` из llama.cpp по
OpenAI-совместимому HTTP API (`POST /v1/chat/completions`). llamadeck — это слой
сверху. Так сделано осознанно: вся тяжёлая C++‑часть остаётся в готовом
llama-server, а наш код — чистый Rust, собирается без боли.

```
llamadeck (Rust) --HTTP/JSON--> llama-server (llama.cpp) --> модель .gguf
   |  веб (axum) + CLI + статистика
```

Весь код общения с движком ИЗОЛИРОВАН в `src/backend.rs`. Это сделано специально,
чтобы будущий переезд на FFI (крейт `llama-cpp-2`, один бинарник без отдельного
llama-server) затронул только этот файл. Не размазывай вызовы движка по другим
модулям.

## Карта файлов

- `src/main.rs` — точка входа, CLI на clap. Подкоманды: `serve` (веб) и `chat` (терминал). Флаг `--model-path` включает авто-запуск движка.
- `src/backend.rs` — слой инференса: `complete()` (полный ответ), `stream()` (потоковый) и `chat_tools()` (запрос с инструментами для агента). Тут и только тут — знание про HTTP-протокол llama-server.
- `src/engine.rs` — авто-запуск `llama-server` дочерним процессом (`tokio::process`) + health-check по `/health` + остановка (`kill_on_drop`). Включается флагом `--model-path`. Запускает движок с `--jinja` (нужно для tool-calling). При переезде на FFI удаляется целиком.
- `src/agent.rs` — агентский режим (идея из проекта chebupelka): каталог инструментов (read_file/list_dir/write_file/run_command), их исполнение, цикл «модель ↔ инструменты» и поток событий для наблюдения. Разрешения: (1) allowlist — агенту предъявляются только разрешённые инструменты; (2) режим прав `PermMode` Auto/Ask — в Ask агент ждёт подтверждения каждого вызова через `Approvals` (карта `oneshot`-каналов, будится из `/api/agent/approve`).
- `src/server.rs` — axum: роуты `/`, `/api/chat`, `/api/chat/stream` (SSE), `/api/stats`, `/api/agent/tools`, `/api/agent/stream` (SSE), `/api/agent/approve`. `AppState` держит `Approvals`. Есть graceful shutdown по Ctrl+C.
- `src/stats.rs` — атомарные счётчики в памяти + `snapshot()`.
- `assets/index.html` — страница чата (встраивается через `include_str!`), стриминг через fetch+reader, опрос статистики раз в секунду.
- `README.md` — инструкция для человека (установка, запуск).

## Как собрать и запустить

Один терминал (llamadeck сам поднимает и гасит движок):
```
cargo run --release -- --model-path models\<модель>.gguf serve   # веб на http://127.0.0.1:3000
cargo run --release -- --model-path models\<модель>.gguf chat     # чат в терминале
```
Полезные флаги авто-запуска: `--llama-bin llama\llama-server.exe` (если не в PATH),
`--ngl 0` (слоёв на GPU; на 4 ГБ начинать с малого), `--ctx 4096`, `--engine-port 8080`.

Два терминала (свой llama-server, старое поведение — без `--model-path`):
```
llama-server.exe -m models\<модель>.gguf --port 8080 -c 4096 -ngl 0
cargo run --release -- serve
cargo run --release -- --upstream http://127.0.0.1:9090 serve   # другой адрес движка
```

## Окружение сборки на этой машине (важно)

- Rust поставлен через rustup (1.96). Дефолт-toolchain — `stable-x86_64-pc-windows-msvc`
  (также установлен `gnu`). Линкер даёт **VC++ Build Tools** (workload VCTools).
- **Smart App Control был выключен** пользователем — иначе он блокировал запуск
  неподписанных build-скриптов (`os error 4551`) и сборка была невозможна. Обратно
  включается только переустановкой Windows.
- В новой сессии cargo/rustc может не быть в PATH — добавляй `$env:USERPROFILE\.cargo\bin`.
- Готовый движок лежит в `llama\llama-server.exe` (CPU-сборка llama.cpp b9831).
  Модели — в `models\` (обе папки в `.gitignore`).

## Нюанс с версиями зависимостей

В `Cargo.toml` стоят пины `clap = "=4.5.4"` и `url = "=2.5.0"`. Это артефакт
сборочной среды, где был **Rust 1.75**. На машине пользователя Rust свежий (rustup),
поэтому пины НЕ нужны. Можно безопасно вернуть `clap = "4"` и убрать строку с `url`,
затем `cargo update`. Сделай это, если потребуются свежие фичи clap.

## Соглашения в коде

- Комментарии и тексты для пользователя — на русском.
- Ошибки — через `anyhow`, с понятными человеку сообщениями (см. `backend.rs`).
- Не добавляй localStorage/внешние UI-фреймворки в `index.html` без необходимости — страница нарочно минимальная и без зависимостей.
- HTTP-only (без TLS): общаемся только с localhost, поэтому reqwest собран без TLS-бэкенда. Не включай TLS без причины.

## Дорожная карта (что пользователь хочет дальше — уточни приоритет)

1. ✅ СДЕЛАНО. Авто-запуск `llama-server` дочерним процессом (`src/engine.rs`, флаг `--model-path`) + health-check.
2. ✅ СДЕЛАНО. Кнопка «Стоп» — отмена генерации. Реализована на фронте (`assets/index.html`) через `AbortController`: обрыв fetch роняет SSE-стрим в axum → reqwest закрывает соединение → llama-server отменяет задачу (`cancel task`). Бэкенд менять не пришлось. Частичный ответ сохраняется в историю.
3. Сохранение истории диалогов на диск.
4. График скорости на странице статистики.
5. Переезд на FFI (`llama-cpp-2`) → один бинарник. Это самый крупный шаг; меняем только `backend.rs`, на Windows понадобится C++‑тулчейн и cmake.

✅ СВЕРХ ПЛАНА (по просьбе пользователя): агентский режим с инструментами,
разрешениями и наблюдением (`src/agent.rs`), вдохновлён проектом chebupelka.
Идеи на развитие: подтверждение каждого вызова (human-in-the-loop), ограничение
путей рабочей папкой (sandbox), стриминг текста агента по токенам.
